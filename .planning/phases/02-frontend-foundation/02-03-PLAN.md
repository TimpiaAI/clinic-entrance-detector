1. Create `frontend/src/api.ts` with typed REST API call wrappers:
   - `apiStartDetector(): Promise<{status: string, pid?: number}>` — POST /api/process/start
   - `apiStopDetector(): Promise<{status: string}>` — POST /api/process/stop
   - `apiDetectorStatus(): Promise<{running: boolean, pid: number|null, exit_code: number|null}>` — GET /api/process/status
   - `apiTestWebhook(): Promise<{status: string}>` — POST /api/test-webhook
   - `apiWakeLockActivate(): Promise<{status: string}>` — POST /api/system/wake-lock
   - `apiWakeLockRelease(): Promise<{status: string}>` — POST /api/system/wake-lock/release
   - Each function uses `fetch()` with appropriate method and headers (`Content-Type: application/json`)
   - Each function handles errors gracefully: try/catch, log error, return a default error response or rethrow
   - Use relative URLs (e.g., `/api/process/start`) so Vite proxy works in dev and same-origin works in prod

2. Create `frontend/src/shortcuts.ts` implementing RESEARCH.md Pattern 3:
   - Type: `ShortcutHandler = () => void | Promise<void>`
   - Module-level `shortcuts: Record<string, ShortcutHandler>` map
   - `registerShortcut(code: string, handler: ShortcutHandler): void` — adds to map
   - `initShortcuts(): void` — attaches a single `document.addEventListener('keydown', ...)` listener
     - In the handler: check `event.code` against shortcuts map
     - If match: call `event.preventDefault()` (CRITICAL: suppresses browser defaults like F3=Find)
     - Then call the handler
   - Export both functions

3. Update `frontend/src/main.ts` to register all 4 shortcuts and call initShortcuts():
   - Import registerShortcut, initShortcuts from shortcuts.ts
   - Import api functions from api.ts
   - Import appState from state.ts (to read detector_running for F2 toggle)

   Register these shortcuts on DOMContentLoaded:

   **F2 — Start/Stop toggle (KEYS-01):**
   ```typescript
   registerShortcut('F2', async () => {
     if (appState.detector_running) {
       await apiStopDetector();
     } else {
       await apiStartDetector();
     }
   });
   ```

   **F3 — Toggle overlay visibility (KEYS-02, FEED-02):**
   ```typescript
   registerShortcut('F3', () => {
     const canvas = document.getElementById('feed-canvas') as HTMLCanvasElement;
     // Toggle opacity between 1 and 0. Do NOT set display:none or remove src.
     // The MJPEG fetch connection must stay open.
     canvas.style.opacity = canvas.style.opacity === '0' ? '1' : '0';
   });
   ```
   CRITICAL: Use opacity, NOT display:none. The fetch-to-canvas loop must keep running even when hidden. Setting display:none could cause the canvas to be garbage collected.

   **F4 — Manual trigger test event (KEYS-03, FEED-06):**
   ```typescript
   registerShortcut('F4', async () => {
     await apiTestWebhook();
   });
   ```

   **Escape — Emergency stop (KEYS-04):**
   ```typescript
   registerShortcut('Escape', async () => {
     await apiStopDetector();
     await apiWakeLockRelease();
   });
   ```

   Call `initShortcuts()` at the end of the DOMContentLoaded handler.
