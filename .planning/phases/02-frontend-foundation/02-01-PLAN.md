  <name>Task 2: MJPEG fetch-to-canvas renderer and WebSocket client with reconnect</name>
  <files>

  <action>
1. Create `frontend/src/feed.ts` implementing MJPEG fetch-to-canvas from RESEARCH.md Pattern 1:
   - Export `startMjpegCanvas(url: string, canvas: HTMLCanvasElement, signal?: AbortSignal): Promise<void>`
   - Use `fetch(url, { signal })` to get a ReadableStream
   - Read chunks into a buffer (Uint8Array)
   - Find JPEG boundaries using SOI marker (0xFF 0xD8) and EOI marker (0xFF 0xD9)
   - For each complete JPEG frame: create Blob, create objectURL, draw to canvas via Image.onload, IMMEDIATELY call URL.revokeObjectURL(blobUrl) after draw (CRITICAL for memory safety)
   - Set canvas.width and canvas.height to match the JPEG frame's naturalWidth/naturalHeight on first frame
   - Handle AbortSignal for clean shutdown
   - Export `stopMjpegCanvas(): void` — calls abort on the AbortController
   - Do NOT use `<img src="/video_feed">` (memory leak — see PITFALLS.md Pitfall 1)

2. Create `frontend/src/ws.ts` implementing WebSocket with exponential backoff from RESEARCH.md Pattern 2:
   - Export `createWsClient(opts: WsOptions): { close: () => void }`
   - WsOptions: { url: string, onMessage: (data: DashboardSnapshot) => void, onStatusChange?: (connected: boolean) => void, baseDelay?: number (default 1000), maxDelay?: number (default 30000), multiplier?: number (default 2) }
   - On open: reset delay to baseDelay, call onStatusChange(true)
   - On message: JSON.parse and pass to onMessage
   - On close: call onStatusChange(false), schedule reconnect
   - On error: close the socket (triggers onclose -> reconnect)
   - Reconnect with delay + 10% random jitter, then multiply delay by multiplier up to maxDelay
   - Export type WsOptions

3. Create `frontend/src/state.ts`:
   - Import DashboardSnapshot from types.ts
   - Export a mutable `appState: DashboardSnapshot` initialized with sensible defaults (fps: 0, entries_today: 0, etc.)
   - Export `updateState(snapshot: DashboardSnapshot): void` that assigns all fields from the incoming snapshot to appState
   - Export `onStateUpdate: ((state: DashboardSnapshot) => void) | null` callback hook that gets called after each updateState

4. Wire everything in `frontend/src/main.ts`:
   - Import startMjpegCanvas from feed.ts
   - Import createWsClient from ws.ts
   - Import { updateState, onStateUpdate } from state.ts
   - On DOMContentLoaded:
     a. Get the canvas element (`feed-canvas`)
     b. Call startMjpegCanvas('/video_feed', canvas) — use relative URL so Vite proxy handles it in dev and same-origin works in prod
     c. Create WebSocket client: `createWsClient({ url: wsUrl, onMessage: updateState })` where wsUrl is constructed as `ws://${window.location.host}/ws` (works in both dev proxy and production same-origin)
     d. Set onStateUpdate to a function that updates any visible state (for now, just console.log the fps)
