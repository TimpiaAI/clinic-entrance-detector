    - Test: detector_status() after start returns {"running": True, "pid": int, "exit_code": None}
    - Test: stop uses psutil tree kill (children enumerated and terminated before parent)
  </behavior>
  <action>
    1. Create `api/__init__.py` (empty package marker).

    2. Create `api/process_manager.py` with:
       - Module-level `_detector_proc: subprocess.Popen | None = None`
       - `_project_root = Path(__file__).resolve().parent.parent`
       - `start_detector() -> dict` function:
         - If `_detector_proc` is not None and `_detector_proc.poll() is None`, return `{"status": "already_running", "pid": _detector_proc.pid}`
         - Otherwise, `subprocess.Popen([sys.executable, str(_project_root / "main.py")], cwd=str(_project_root), stdout=subprocess.PIPE, stderr=subprocess.PIPE)`
         - Return `{"status": "started", "pid": _detector_proc.pid}`
       - `stop_detector() -> dict` function:
         - If not running, set `_detector_proc = None`, return `{"status": "not_running"}`
         - Use psutil tree kill: `parent = psutil.Process(pid)`, `children = parent.children(recursive=True)`, terminate all children then parent, `psutil.wait_procs([parent] + children, timeout=5)`, force-kill survivors
         - Wrap in try/except psutil.NoSuchProcess
         - Set `_detector_proc = None`, return `{"status": "stopped", "pid": pid}`
       - `detector_status() -> dict` function:
         - If `_detector_proc is None`: return `{"running": False, "pid": None, "exit_code": None}`
         - If `poll() is not None`: return `{"running": False, "pid": ..., "exit_code": poll}`
         - Else: return `{"running": True, "pid": ..., "exit_code": None}`
       - `router = APIRouter(prefix="/api/process", tags=["process"])` with three route handlers that call the functions above:
         - `POST /start` -> start_detector()
         - `POST /stop` -> stop_detector()
         - `GET /status` -> detector_status()

    3. Create `tests/test_process_manager.py`:
       - Use `unittest.mock.patch` to mock `subprocess.Popen` and `psutil.Process`
       - Test all behaviors listed above
       - Do NOT actually spawn main.py in tests -- mock the subprocess
       - Use `importlib.reload` or reset `_detector_proc` between tests to avoid state leakage

    Pattern reference: See RESEARCH.md "Pattern 2: Process Manager with Module-Level Singleton" for the complete implementation pattern. Follow it exactly.

    CRITICAL: Set `cwd=str(_project_root)` in Popen to avoid Pitfall 5 (wrong working directory).
    CRITICAL: Use psutil tree kill, NOT just `process.terminate()` to avoid Pitfall 1 (zombie processes holding camera on Windows).
