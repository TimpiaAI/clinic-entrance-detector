1. Create `frontend/src/video.ts` implementing the video playback module with these exports:

**Constants:**
```typescript
const IDLE_VIDEO = 'video1.mp4';
const INSTRUCTION_SEQUENCE = [
  'video2.mp4',  // greeting
  'video3.mp4',  // ask name
  'video6.mp4',  // ask (question)
  'video7.mp4',  // ask CNP
  'video8.mp4',  // ask email
  'video4.mp4',  // farewell
  'video5.mp4',  // final
];
```
This EXACT sequence matches controller.py's workflow() function.

**State:**
```typescript
type VideoState = 'idle' | 'playing_sequence' | 'hidden';
let state: VideoState = 'hidden';
let sequenceIndex = 0;
let userGestureReceived = false;
```

**Core functions:**

`playVideo(filename: string, opts?: { loop?: boolean }): Promise<void>` — Reuse the SINGLE `<video id="video-overlay">` element. Set `video.loop`, set `video.src = /api/videos/${filename}`, call `video.load()`, set `video.style.opacity = '1'`, then `await video.play()` with try/catch for autoplay NotAllowedError. Log errors to console. NEVER create/destroy video elements (Chromium memory leak -- Chromium issue 41462045).

`hideVideo(): void` — Set `video.style.opacity = '0'`, `video.pause()`, `video.removeAttribute('src')`, `video.load()` (W3C spec: releases buffered data when src removed and load() called).

`startIdleLoop(): void` — Set state to 'idle', reset sequenceIndex to 0, call `playVideo(IDLE_VIDEO, { loop: true })`. Note: `onended` does NOT fire when `loop=true` (MDN confirmed) so idle state is naturally stable.

`startInstructionSequence(): void` — Guard: `if (state === 'playing_sequence') return;` (prevents re-trigger during active sequence -- Pitfall 4 from RESEARCH.md). Set state to 'playing_sequence', sequenceIndex to 0, call `playNextInSequence()`.

`playNextInSequence(): void` — If sequenceIndex >= INSTRUCTION_SEQUENCE.length, call startIdleLoop() and return. Otherwise get filename from INSTRUCTION_SEQUENCE[sequenceIndex], increment sequenceIndex, call `playVideo(filename, { loop: false })`.

`onUserGesture(): void` — If `userGestureReceived` is true, return. Set `userGestureReceived = true`. Set `video.muted = false`. This unmutes after operator's first interaction (F2 keypress). Until then, videos play muted for autoplay compatibility.

`getVideoState(): VideoState` — Returns current state for external consumers.

**Event wiring (inside an `initVideo()` function):**

Get the video element: `document.getElementById('video-overlay') as HTMLVideoElement`.

Add `ended` event listener: `video.addEventListener('ended', () => { if (state === 'playing_sequence') { playNextInSequence(); } });`

This is the core of VCTL-05 -- onended-driven transitions, not timer-based.

Export `initVideo`, `startIdleLoop`, `startInstructionSequence`, `hideVideo`, `onUserGesture`, `getVideoState`.
