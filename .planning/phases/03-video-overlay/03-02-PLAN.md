const VIDEO_LABELS: Record<string, string> = {
  'video2.mp4': RO.VIDEO_GREETING,
  'video3.mp4': RO.VIDEO_ASK_NAME,

Add `checkForPersonEntered(eventLog: EventLogEntry[]): void`:
- The event_log array is newest-first (Python deque appendleft).
- Find the latest entry where `e.event === 'person_entered'` using `eventLog.find(...)`.
- If none found, return.
- If `latest.timestamp === lastPersonEnteredTimestamp`, return (already handled).
- Set `lastPersonEnteredTimestamp = latest.timestamp`.
- Call `startInstructionSequence()`. The existing guard inside startInstructionSequence prevents re-trigger during active sequence.

CRITICAL for Pitfall 5 (stale events on reconnect): The timestamp comparison naturally handles reconnects. On reconnect, the full event_log is re-sent, but the latest person_entered event has the same timestamp as before. Since we track by timestamp (not by count), no phantom trigger occurs. The only way a trigger fires is if a genuinely NEW person_entered event appears with a timestamp strictly different from the last processed one.

2. Add marquee text overlay functions to `frontend/src/video.ts`:

`showMarquee(text: string): void`:
- Get `#text-overlay` element.
- Get its `.marquee-content` child span.
- Set `span.textContent = text`.
- Set `overlay.style.opacity = '1'`.

`hideMarquee(): void`:
- Get `#text-overlay` element.
- Set `overlay.style.opacity = '0'`.

3. Integrate marquee into the video sequence:

Modify `playNextInSequence()` to show the corresponding marquee label when each instructional video starts:
- After calling `playVideo(filename, { loop: false })`, look up `VIDEO_LABELS[filename]`.
- If a label exists, call `showMarquee(label)`.
- If no label (shouldn't happen but defensive), call `hideMarquee()`.

Modify `startIdleLoop()` to hide the marquee when returning to idle:
- Call `hideMarquee()` at the beginning of `startIdleLoop()`.

4. Export the new functions: add `checkForPersonEntered`, `showMarquee`, `hideMarquee` to exports.

5. In `frontend/src/main.ts`:

This wiring means: every 0.5s when a WebSocket snapshot arrives, the video module checks if a new person_entered event appeared and triggers the sequence if so.
